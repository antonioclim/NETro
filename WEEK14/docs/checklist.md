# Checklist Cadru Didactic — Săptămâna 14

## Înainte de curs/seminar

### Pregătire tehnică
- [ ] VM de demonstrație pregătită și testată
- [ ] Starterkit funcțional (rulat `make run-demo` cu succes)
- [ ] Backup: capturi pcap pregătite (dacă demo-ul eșuează live)
- [ ] Proiector/ecran funcțional

### Materiale
- [ ] Slide-uri/notițe pentru recapitulare
- [ ] Rubrică de evaluare proiecte (printată sau digitală)
- [ ] Lista echipelor și ordinea prezentărilor
- [ ] Întrebări de apărare pregătite

### Administrative
- [ ] Lista studenților prezenți
- [ ] Formular de notare proiecte

## În timpul cursului

### Recapitulare (45 min)
- [ ] Prezintă „big picture" (modelul complet al comunicării)
- [ ] Checklist de diagnostic (pe tablă/slide)
- [ ] Mini-demo 1: TCP handshake în pcap
- [ ] Mini-demo 2: Connection refused vs timeout
- [ ] Întrebări de control (2-3 pe parcurs)

### Legătura cu proiectul (15 min)
- [ ] Ce așteptăm la prezentare
- [ ] Exemple de întrebări de apărare
- [ ] Rubrică explicată

### Q&A final (15 min)
- [ ] Întrebări despre examen
- [ ] Clarificări despre proiect

---

## În timpul seminarului

### Setup (10 min)
- [ ] Verifică că studenții au acces la mediu
- [ ] Stabilește ordinea prezentărilor
- [ ] Explică regulile (timp, Q&A)

### Prezentări (10-12 min/echipă)
- [ ] Cronometrează
- [ ] Notează pe rubrică
- [ ] Pune 2-3 întrebări de apărare
- [ ] Feedback scurt (opțional, la final)

### Închidere
- [ ] Anunță rezultatele (dacă sunt gata)
- [ ] Reamintește: accesul la examen
- [ ] Linkuri utile pentru pregătire

---

## După seminar

### Evaluare
- [ ] Finalizează notele pe rubrică
- [ ] Introduce în catalog
- [ ] Marchează studenții care NU au prezentat

### Follow-up
- [ ] Email cu rezultatele (opțional)
- [ ] Resurse pentru pregătirea examenului
- [ ] Feedback pentru îmbunătățirea kit-ului

---

## Capcane frecvente și soluții

| Capcană | Soluție |
|---------|---------|
| Demo nu pornește | Folosește capturile pregătite |
| Studenții nu au VM | Demonstrație pe VM cadru |
| Întrebări prea grele | Ajustează la nivelul real |
| Timp depășit | Oprește politicos, notează parțial |
| Conflicte de port | `make clean` înainte de fiecare demo |
